## Summary

Build the SKILL.md agent instruction file that guides the LLM to generate citation-backed RCSA control narratives from pre-processed mapped evidence. The SKILL.md defines the LLM's workflow: receive structured mapped evidence for four compliance controls (Access Control, Change Management, Data Quality, Incident Handling), generate 3–5 sentence auditor-friendly narratives with inline citations referencing exact artifact registry identifiers, explicitly flag Gaps when evidence is missing or insufficient, and produce a summary table — all following output templates defined inline (later formalized by F4 Assets). The deterministic pipeline (F6 — Scripts) handles all input validation, evidence mapping, and post-processing; the SKILL.md governs only the LLM reasoning step.

🆕 The SKILL.md must also define the sequence of tool calls the agent should make: validate inputs → build registry → map evidence → generate narratives → validate citations (FR-010). This orchestration logic is a core architectural element of the agent skill.

## Technical Context

**Language/Version**: Natural language (Markdown) — SKILL.md is a prompt instruction file, not executable code
**Primary Dependencies**: Claude Agent Skills runtime environment. Note: F6 (Scripts) and F4 (Assets) are downstream features that will be built to support and formalize what F5 defines. During F5 development, output structure is defined inline and sample inputs are created manually for testing
**Storage**: N/A — SKILL.md is a static instruction file; output files (`rcsa_control_narratives.md`, `validation_report.md`) are written by F6 (Scripts)
**Testing**: Manual validation — run SKILL.md with sample mapped evidence inputs and verify output against acceptance scenarios in spec; automated structural validation handled by F6 (Scripts)
**Target Platform**: Claude Agent Skills (Cursor IDE integration)

**Project Type**: Agent skill definition (prompt engineering / instruction authoring)
**Performance Goals**: LLM generates complete output for all 4 controls in a single invocation; ≥ 90% structural consistency across repeated runs
**Constraints**: SKILL.md must instruct the LLM to use only provided evidence (no external knowledge); must never imply compliance without proof (Constitution Principle 1); must define output structure inline (later formalized into templates by F4); must instruct the LLM to cite artifacts using exact identifiers from the artifact registry (FR-007)
**Scale/Scope**: Single SKILL.md file covering 4 compliance controls; designed for demo-scale repositories (not enterprise-scale)

## 🆕 Edge Case Awareness (Grenade Map)

> **"You can't fix everything. But you can know what might break."**

### Tier 1 — Handled in SKILL.md (Demo-Critical)

| Edge Case | Handling | Phase |
|---|---|---|
| Empty evidence for all 4 controls | SKILL.md instructs: produce complete Gap report for all controls, never return empty response | Phase 2 (I-7) |
| LLM generates narratives without citations | SKILL.md hard rule: every evidence-referencing sentence MUST include `[artifact-id]` citation | Phase 2 (I-3) |
| LLM invents artifact IDs not in input | SKILL.md hard rule: ONLY cite identifiers present in mapped evidence input | Phase 2 (I-3) |
| False compliance narrative (implies compliance without proof) | SKILL.md hard rule: when in doubt, flag Gap (Constitution Principle 1) | Phase 2 (I-4) |
| Summary table contradicts narrative body | SKILL.md instruction: generate summary table AFTER narratives, derived from results | Phase 2 (I-5) |
| Fewer than 4 controls in output | SKILL.md instruction: MUST address all 4 controls; missing input = Gap flag | Phase 2 (I-3, I-4) |

### Tier 1 — Handled by F6 Scripts (Infrastructure)

| Edge Case | Handling | Owner |
|---|---|---|
| LLM rate limit exceeded (429) | Retry with exponential backoff (1s → 2s → 4s → 8s, max 3 retries), then graceful degradation | F6 |
| LLM timeout | 120s timeout, retry once, then graceful degradation | F6 |
| LLM malformed response | Validate response structure before writing output files; retry once on failure | F6 |
| LLM empty/null response | Explicit empty check — empty ≠ "no gaps found"; log failure, retry, degrade | F6 |

### Tier 2 — Known Limitations (Documented, Not Built)

| Edge Case | Why Deferred | Future Handling |
|---|---|---|
| Enterprise-scale input (hundreds of artifacts) | Demo-scale scope per Constitution Principle 3 | Implement chunking and pagination |
| Non-English evidence | Demo uses English Kaggle dataset; Synchrony handoff responsibility | Multi-language support in production |
| Evidence for controls outside 4-control scope | Spec scopes to exactly 4 controls | Extend control definitions in SKILL.md |
| Contradictory evidence across artifacts | Not part of happy path demo | Flag conflicts rather than choosing sides |
| Concurrent skill invocations | Demo is single-user | Add concurrency controls in production |
| Model version drift | Mitigate by testing before demo day | Pin model version in production |
| Ambiguous evidence across multiple controls | Adds complexity that could destabilize happy path | Cite conservatively and note ambiguity |

## 🆕 Demo Script (5-Step Flow)

> Based on Tier 1 features. Total time: ~4 minutes.

1. **Set the Stage (30s)** — Show the problem: scattered compliance artifacts, manual audit effort
2. **Show the Input (45s)** — Display pre-processed mapped evidence with artifact IDs; highlight that one control has no evidence (sets up gap detection)
3. **Run the Skill (60s)** — Invoke SKILL.md in Claude Agent Skills; narrate the workflow as it processes
4. **Show Summary Table (30s)** — Three controls ✅ Evidence, one ❌ Gap — auditor sees compliance posture at a glance
5. **Walk Through Output (60s)** — Show one narrative with inline citations, then the Gap flag: "The system never implies compliance without proof"

**Backup Plan**: Pre-generated output saved during Phase 3 (V-5). If Claude API fails during live demo, switch to saved output.
